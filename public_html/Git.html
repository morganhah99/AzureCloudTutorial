Introduction:

Git is a free and open source distributed version
control system designed to handle everything from small to very
large projects with speed and efficiency.

Installation:

  1. Install the Git client with this link
  2. Create a Github account here

Git Configuration:

To start using Git on your computer you must first enter your
credentials to identify yourself as the author of your work

  1. Open up Git Bash terminal
  2. Enter git config --global user.name "your_username"
  3. Enter git config --global user.email "your_email"
  4. To check the config enter git configuration --global --list

How to make a repository

A Git repository is a collection of files of various different versions of a project.
These files are imported from the repository into the local server of the user for further updations and modifications in the content of the file.
A version control system is used to create these versions and store them in a specific place called a repository.

  1. Open the Git Bash terminal
  2. Navigate to the directory of where you want the repository
  3. Enter touch README.md
  4. Enter git init
  5. Enter git add README.md
  6. Enter git commit -m "first commit"
  7. Enter git remote add origin git@github.com:your_username/.git
  8. Enter git push -u origin master

List of helpful Git commands

  • git clone [url] (Obtain a repository from an exisiting URL)
  • git add [file] (Adds file to the staging area)
  • git commit -m "[commit message]" (Snapshots the file permanently in the version history)
  • git commmit -a (Commits any files you've added with the git add commit and also commits and files you've changed since then)
  • git status (Lists all files that have to be committed)
  • git rm [file] (Deletes the file from your working directory and stages the deletion
  • git branch (lists all the local branches in the local repository
  • git branch [branch_name] (Creates new branch)
  • git checkout [branch_name] (Switches to another branch)
  • git checkout -b [branch_name] (creates new branch but also switches to it)
  • git merge [branch_name] (merges the specific branch's history into the current branch)
  • git remote add [variable_name] [remote_server_link] (Used to connect your local repository to the remote server)
  • git push [variable_name] master (Sends the commmited changes of master branch to your remote repository)
  • git pull [repository_name] (Fetches and merges changes on the remote server to your working directory)
  • git init [repository_name] (creates repository in curent directory)
